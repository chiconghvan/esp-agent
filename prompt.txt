Các câu hỏi mà tôi có thể truy vấn là:
"Ngày mai tôi có lịch gì?"
"Tuần này có lịch nào không?"
"Có báo cáo nào đến hạn không?"

"Tôi đã hoàn thành báo cáo quý 1"

thay đổi trạng thái done task khi hoàn thành.
Thì project cần trả lời chính xác theo câu hỏi của tôi.
Ý tưởng workflow:
Với mỗi tin nhắn tôi nhắn vào bot tele: Có 7 action cốt lõi
🟢 1️⃣ CREATE_TASK 
Tạo task mới.
Ví dụ:
"Nhắc tôi nộp báo cáo quý 1 vào 31/3"
"Thêm lịch họp thứ 6 tuần này"

🔵 2️⃣ QUERY_BY_TIME_RANGE
Hỏi lịch theo khoảng thời gian.
Bao gồm:
Ngày mai
Tuần này
Tháng này
Hôm nay
Ví dụ:
"Ngày mai có gì?"
"Tuần này có lịch nào không?"
ESP32 chỉ filter theo time range.
🟣 3️⃣ QUERY_BY_TYPE
Hỏi theo loại task.
Ví dụ:
"Có báo cáo nào đến hạn không?"
"Có công việc nào pending?"
Filter theo:
type
status
due_time
🟡 4️⃣ QUERY_SEMANTIC
Tìm task theo nghĩa.
Ví dụ:
"Thời hạn báo cáo quý 1?"
"Lịch công tác Hà Nội khi nào?"
Dùng embedding search.
🟠 5️⃣ COMPLETE_TASK
Đánh dấu hoàn thành.
Ví dụ:
"Tôi đã hoàn thành báo cáo quý 1"
"Xong việc họp phòng kế toán"
Update:
status = done
completed_at = now
Nếu repeat → tạo bản tiếp theo.
🔴 6️⃣ DELETE_TASK
Xóa task.
Ví dụ:
"Hủy lịch họp ngày mai"
"Xóa báo cáo quý 1"
Có thể:
Hard delete
Hoặc set status = cancelled
🟤 7️⃣ UPDATE_TASK
Sửa task.
Ví dụ:
"Dời lịch họp sang thứ 7"
"Báo cáo quý 1 đổi hạn sang 2/4"
Update:
start_time
due_time
reminder

- B2: Query Parser (chỉ chạy khi intent rõ ràng)
Mỗi intent có JSON schema khác nhau.
Sau khi có intent, hãy tiếp tục đưa intent và request của user vào prompt mới để phân tích yêu cầu của user
Đầu tiên cần phân tích xem request của user có bao nhiêu điều kiện lọc, ví dụ: input là “Tôi có những task báo cáo lặp lại hàng tháng nào” thì hệ thống cần phải query type=report, time=trong tuần này, status=pending
yêu cầu json trả về có định dạng 
{
  "field_count": number,
  "filters": [ 
    {
        "field": "field_name; id, type, title, status, due_time, start_time, reminder, repeat, repeat_interval, notes",
        "operator": "equals | contains | before | after | between | in | is_not_null | is_null |...",
        "value": value,
        "confidence": 0.0-1.0 <nếu confidence < 0.8 thì hỏi lại user>
    }
  ]          <- tự đề xuất các trường thông tin, sửa đổi đối với từng phần tử của mảng filters 
}
Sử dụng tìm kiếm vector đối với các trường hợp request có nội dung cụ thể liên quan đến tiêu đề, nội dung một cách cụ thể thì sử dụng tìm kiếm vector trong các vector đã lưu, nếu có nhiều vector có ngữ nghĩa tương đồng (confidence gần bằng nhau với request thì liệt kê tất ra cho user xem.
